## Key Insights

- `npx expo install expo@latest` handles SDK + RN + React version bumps
- `npx expo install --fix` auto-resolves compatible versions for all expo-* packages
- Reanimated v4 is MANDATORY for SDK 55 (v3 incompatible)
- react-native-worklets is a required peer dep for reanimated v4
- Use `--legacy-peer-deps` for NativeWind / testing-library peer conflicts

## Requirements

### Functional
- All packages updated to SDK 55 compatible versions
- No unresolved peer dependency errors blocking builds

### Non-functional
- package-lock.json must be regenerated cleanly
- No manual version pinning unless expo-doctor flags issues

## Architecture

No architecture changes. Pure dependency version bumps.

- `package.json` - all version changes happen here
- `package-lock.json` - regenerated by npm install

## Implementation Steps

### Step 1: Upgrade Expo SDK

```bash
npx expo install expo@latest
```

Expected: `expo` bumps to `~55.0.0`, pulls `react-native` 0.83.2 and `react` 19.x.

### Step 2: Fix All Expo Dependencies

```bash
npx expo install --fix
```

Expected auto-resolution:
- `expo-router` -> ~5.x
- `expo-constants` -> compatible version
- `expo-device` -> compatible version
- `expo-font` -> compatible version
- `expo-linking` -> compatible version
- `expo-notifications` -> compatible version
- `expo-splash-screen` -> compatible version
- `expo-status-bar` -> compatible version
- `react-native-reanimated` -> ~4.1.0+
- `react-native-gesture-handler` -> ~2.30.0
- `react-native-safe-area-context` -> compatible version
- `react-native-screens` -> compatible version
- `@types/react` -> ~19.x

### Step 3: Run Expo Doctor (First Pass)

```bash
npx expo-doctor
```

Document ALL warnings/errors. Common issues:
- Version mismatches not caught by `--fix`
- Deprecated packages
- Incompatible native module versions

### Step 4: Install react-native-worklets

```bash
npx expo install react-native-worklets
```

Required peer dep for reanimated v4. Without this, reanimated crashes at runtime.

### Step 5: Resolve Peer Deps

```bash
npm install --legacy-peer-deps
```

Needed if NativeWind or @testing-library/react-native has unresolved React 19 peer deps.

### Step 6: Verify package.json

Check these targets in `package.json`:

| Package | Expected Version |
|---------|-----------------|
| expo | ~55.0.x |
| react | 19.x |
| react-native | 0.83.x |
| react-native-reanimated | ~4.1.x |
| react-native-worklets | (any stable) |
| expo-router | ~5.x |
| @types/react | ~19.x |
